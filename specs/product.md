# プロダクト仕様書 - Ninjo

## 概要

Ninjo は勉強計画を立て、AIがその妥当性を判断し、毎日の進捗をトラッキングする学習管理アプリ。
書籍のページ数ベースで計画を立て、実績データからAIがペースを推定。
本を読み終えた時には、メモからAIが学習サマリーを自動生成する。

## ターゲットユーザー

- **ペルソナ**: 大学生〜若手社会人。資格取得・スキルアップを計画的に進めたい人
- **課題**: 計画は立てるけど続かない。進捗が可視化されず漠然と不安になる
- **求めているもの**: 「今のペースで間に合うか」を正直に教えてくれて、遅れた時に一緒にリカバリしてくれる存在

## コア機能

### F1. 計画作成 + AIレビュー

| 項目 | 内容 |
|---|---|
| 概要 | 教材名・総ページ数・期限を入力して学習計画を作成する |
| AIの役割 | 可処分時間と（あれば）過去実績から、1日あたり必要ページ数を算出し、妥当性をレビュー |
| 入力 | 教材名、総ページ数、目標期限 |
| 出力 | AIレビュー結果（1日の必要ページ数、現実的かの判断、アドバイス） |

### F2. デイリータスク

| 項目 | 内容 |
|---|---|
| 概要 | 全計画を横断して「今日やること」を1画面に表示 |
| AIの役割 | 計画から今日分のタスクを自動提案 |
| ユーザー操作 | AIの提案をそのまま使う or 自分で調整 |

### F3. タスク完了 + メモ

| 項目 | 内容 |
|---|---|
| 概要 | チェックボックスで完了。完了時に学んだ内容をメモする |
| メモ | 完了タイミングで入力。自由テキスト |
| データ活用 | メモは全て保存され、F5の学習サマリーの素材になる |

### F4. 進捗分析 + 軌道修正

| 項目 | 内容 |
|---|---|
| 概要 | 過去実績からペースを分析し、遅れを検知してリスケ提案 |
| AIの役割 | 実績が溜まったら「1時間あたりXページ」を推定。遅れ時にリスケ案を生成 |
| 表示 | 計画ごとの進捗率、予定 vs 実績の差分 |

### F5. 学習サマリー

| 項目 | 内容 |
|---|---|
| 概要 | 本が完了した時、全メモからAIが学習サマリーを自動生成 |
| 出力 | 習得した知識、成長ポイント、復習推奨箇所 |
| トリガー | 全ページ完了時に自動生成 |

### F6. 可処分時間設定

| 項目 | 内容 |
|---|---|
| 概要 | 曜日ごとに勉強できる時間を設定 |
| 入力 | 月〜日それぞれの時間（h単位） |
| 用途 | AIの計画レビュー・デイリータスク配分の基礎データ |

## MVP スコープ（v1.0）

**含む:**
- F6: 可処分時間設定
- F1: 計画作成 + AIレビュー
- F2: デイリータスク
- F3: タスク完了 + メモ
- ユーザー認証（メール + パスワード）

**v1.1 以降:**
- F4: 進捗分析 + 軌道修正
- F5: 学習サマリー
- AIによるペース推定（実績データ蓄積後）
- 通知・リマインダー

## 画面構成（MVP）

```
[ログイン / サインアップ]
        ↓
[ダッシュボード]  ← 今日のタスク一覧（全計画まとめ）
        ↓
[計画一覧]       ← 登録中の学習計画 + 各進捗率
        ↓
[計画作成]       ← 教材・ページ数・期限入力 → AIレビュー → 確定
        ↓
[計画詳細]       ← 進捗 + メモ一覧
        ↓
[設定]           ← 曜日別の可処分時間
```

## 成功指標

| 指標 | 目標 |
|---|---|
| 計画完了率 | 作成した計画の50%以上が完了 |
| デイリータスク消化率 | 平均60%以上 |
| メモ記入率 | タスク完了時の70%以上でメモあり |
| 継続率 | 7日後リテンション40% |
