# F7: アバター管理 - 機能仕様書

## 概要

ドット絵プリセットから自分の分身（アバター）を選択する。
アバターはホーム画面の中央に表示され、蓄積されたスキルと共にユーザーの成長を可視化する。

## ユーザーストーリー

> ユーザーとして、ドット絵のアバターを選んで自分の分身として設定し、
> 勉強を通じてこの分身にスキルが積み上がっていくのを見たい。

## API エンドポイント

### GET /api/v1/avatar

現在のアバター設定とスキルサマリーを取得する。
未設定の場合はデフォルトアバターを返す。

**Response (200):**
```json
{
  "avatar_preset_id": "preset_01",
  "skill_count": 12,
  "skill_categories": [
    { "category": "プログラミング", "count": 5 },
    { "category": "インフラ", "count": 4 },
    { "category": "セキュリティ", "count": 3 }
  ]
}
```

**エラー:**
- 401: 未認証

### PUT /api/v1/avatar

アバターのプリセットを設定/変更する。

**Request:**
```json
{
  "avatar_preset_id": "preset_01"
}
```

**Response (200):**
```json
{
  "avatar_preset_id": "preset_01"
}
```

**バリデーション:**
- avatar_preset_id: 有効なプリセットIDであること

**エラー:**
- 400: 無効なプリセットID
- 401: 未認証

## データ設計

- users テーブルに `avatar_preset_id` カラムを追加（string, nullable）
- null の場合はデフォルトアバター（"preset_01"）として扱う
- スキルデータは F9 の skills テーブルから集約して表示

## アバタープリセット

MVP では 8-12 種のドット絵キャラクターを用意:
- プリセットIDは "preset_01" 〜 "preset_12"
- 画像アセットは `frontend/public/avatars/` に配置
- 性別・属性に偏らないニュートラルなデザイン

## 認証

- GET/PUT ともに認証必須

## ビジネスルール

- アバターはいつでも設定画面から変更可能
- アバターを変更してもスキルデータは保持される
- MVPスコープ外: アニメーション、着せ替え、レベルシステム
