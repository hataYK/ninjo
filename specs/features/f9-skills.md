# F9: スキル抽出・蓄積 - 機能仕様書

## 概要

タスク完了時に書いた学習メモからAIがスキルを抽出し、アバターのプロフィールに蓄積する。
ユーザーはAIの提案を確認・編集し、カテゴリを選択できる（ハイブリッド方式）。
蓄積されたスキルがアバターの「成長」そのものとなる。

## ユーザーストーリー

> ユーザーとして、学習メモを書くと自分が何を学んだかがスキルとして可視化され、
> アバターのプロフィールに蓄積されていくのを見たい。

## API エンドポイント

### POST /api/v1/tasks/{task_id}/extract-skills

タスクのメモからAIがスキルを抽出する。保存はしない（提案のみ）。

**Response (200):**
```json
{
  "suggested_skills": [
    { "name": "AWS: VPC設計", "category": "インフラ" },
    { "name": "AWS: サブネット構成", "category": "インフラ" },
    { "name": "AWS: セキュリティグループ", "category": "セキュリティ" }
  ]
}
```

**エラー:**
- 400: メモが空（スキル抽出不可）
- 401: 未認証
- 404: タスクが存在しない

### GET /api/v1/skills

ユーザーの全スキル一覧を取得する。

**Query Parameters:**
- `category` (optional): カテゴリでフィルタ

**Response (200):**
```json
{
  "skills": [
    {
      "id": "uuid",
      "name": "AWS: VPC設計",
      "category": "インフラ",
      "source": "ai",
      "task_id": "uuid",
      "created_at": "2026-05-10T10:00:00Z"
    }
  ],
  "total_count": 12
}
```

### POST /api/v1/skills

スキルを手動追加する。

**Request:**
```json
{
  "name": "Go: goroutine",
  "category": "プログラミング",
  "task_id": "uuid"
}
```

- `task_id` は optional（タスクに紐付けない手動追加も可能）

**Response (201):**
```json
{
  "id": "uuid",
  "name": "Go: goroutine",
  "category": "プログラミング",
  "source": "manual",
  "task_id": null,
  "created_at": "2026-05-10T10:00:00Z"
}
```

### PUT /api/v1/skills/{skill_id}

スキルの名前やカテゴリを更新する。

**Request:**
```json
{
  "name": "Go: goroutineとチャネル",
  "category": "プログラミング"
}
```

**Response (200):** 更新後のスキルオブジェクト

### DELETE /api/v1/skills/{skill_id}

スキルを削除する。

**Response (204):** No Content

## AI スキル抽出のルール

### プロンプト設計

```
以下の学習メモからスキルを抽出してください。

教材: {plan_title}
メモ:
{memo_text}

ルール:
- 具体的で履歴書に書けるレベルのスキルのみ抽出する
- 「勉強した」「理解した」のような曖昧なものは除外
- 形式: "分野: 具体的なスキル名"（例: "AWS: VPC設計"）
- 最大5個まで
- 各スキルにカテゴリを付与する

出力形式（JSON）:
[{"name": "分野: スキル名", "category": "カテゴリ名"}]
```

### 抽出品質の担保

- メモが短すぎる場合（目安: 20文字未満）はスキル抽出をスキップし、手動追加を案内
- 同一ユーザーの既存スキルと重複する場合はAI提案から除外
- ユーザーは提案を自由に編集・削除・追加できる

## データ設計

### skills テーブル

| カラム | 型 | 説明 |
|---|---|---|
| id | UUID | PK |
| user_id | UUID | FK → users |
| task_id | UUID (nullable) | FK → daily_tasks |
| name | string | スキル名 |
| category | string | カテゴリ名 |
| source | string | "ai" or "manual" |
| created_at | timestamp | 作成日時 |

### インデックス

- user_id（スキル一覧取得）
- user_id + category（カテゴリフィルタ）
- user_id + name（重複チェック、UNIQUE）

## 認証

- 全エンドポイント認証必須
- 自分のスキルのみ操作可能

## ビジネスルール

- スキルは計画やタスクを削除しても保持される（学びは消えない）
- 同一ユーザー内でスキル名の重複は不可
- source は作成時に自動設定（AI抽出="ai"、手動追加="manual"）
- AI抽出後にユーザーが編集した場合も source は "ai" のまま
